# 📂 SDIR (Shai Direct Interface) 기술 매뉴얼
Skript-Base File Manager and File Editor SDIR v0.9
Copyright 2026. Star^N Studio All rights reserved.

SDIR은 마인크래프트 서버 내에서 파일과 디렉토리(폴더)를 직접 제어하고, 코드를 실시간으로 편집할 수 있는 최고 관리자 전용 CLI(명령줄 인터페이스) 시스템입니다.

## ⚠️ 시스템 주의 사항 (필독)
* **복구 불가:** SDIR을 통해 삭제되거나 덮어씌워진 파일은 휴지통으로 이동하지 않으며 **영구적으로 소멸**됩니다. 명령어 실행 전 경로를 반드시 재확인하십시오.
* **보안 권한:** 서버의 최고 관리자(`OP`) 권한을 가진 유저만 시스템에 접근할 수 있습니다.

---

## 📌 1. 문법 기호 및 경로 작성 규칙

### 기호 약속
* **`(값)` : 필수 입력값** (반드시 입력해야 명령어가 작동합니다.)
* **`[값]` : 선택 입력값** (상황에 따라 생략할 수 있습니다.)
* **`(A | B)` : 필수 선택 입력값** (반드시 A와 B 중 하나를 선택하여 입력해야 합니다.)

### 경로 작성 규칙
* **띄어쓰기 지양:** 파일/폴더명 및 경로에 띄어쓰기가 포함되는 것을 가능한 지양해야 합니다.
* **확장자 필수:** 대상을 `file`로 지정한 경우, 반드시 끝에 확장자(`.sk`, `.yml` 등)를 포함해야 합니다.
* **하위 디렉토리 구분:** 슬래시(`/`)를 사용하여 상위 폴더와 하위 폴더를 구분합니다. *(예: `plugins/Skript/scripts/test.sk`)*

## 🛠️ 2. 파일 및 폴더 제어 명령어

### 2-1. 생성 (Create)
지정한 경로에 빈 파일이나 폴더를 생성합니다. (상위 폴더가 없다면 자동 생성됨)
* **정식 문법:** `/sdir create (file | dir) (생성할_경로)`
* **단축 문법:** `/sdir c (f | d) (생성할_경로)`

### 2-2. 삭제 (Delete)
파일이나 폴더를 영구적으로 삭제합니다.
* **정식 문법:** `/sdir delete (file | dir) (삭제할_경로)`
* **단축 문법:** `/sdir d (f | d) (삭제할_경로)`
* *주의:* `dir` 삭제 시 내부의 모든 하위 항목이 함께 삭제됩니다.

### 2-3. 이름 변경 (Rename)
파일이나 폴더의 이름을 변경합니다.
* **정식 문법:** `/sdir rename (file | dir) (원본_경로) (새_이름)`
* **단축 문법:** `/sdir r (f | d) (원본_경로) (새_이름)`
* *주의:* `(새_이름)`에는 전체 경로를 제외한 순수 파일/폴더명만 입력합니다.

### 2-4. 이동 (Move) 및 복사 (Copy)
* **정식 문법:**
  * 이동: `/sdir move (file | dir) (원본_경로) (도착_경로)`
  * 복사: `/sdir copy (file | dir) (원본_경로) (도착_경로)`
* **단축 문법:**
  * 이동: `/sdir m (f | d) (원본_경로) (도착_경로)`
  * 복사: `/sdir (p | co) (f | d) (원본_경로) (도착_경로)`

## 🌐 3. 네트워크 및 압축 명령어

### 3-1. 다운로드 (Download)
인터넷 웹상의 코드나 파일을 서버 로컬에 즉시 저장합니다.
* **정식 문법:** `/sdir download (URL_주소) (저장될_파일경로)`
* **단축 문법:** `/sdir (l | do) (URL_주소) (저장될_파일경로)`

### 3-2. 압축 (Zip) 및 해제 (Unzip)
* **정식 문법:**
  * 압축: `/sdir zip (file | dir) (압축할_경로) (생성될_zip_경로)`
  * 해제: `/sdir unzip (풀어줄_zip_경로) (도착_폴더_경로)`
* **단축 문법:**
  * 압축: `/sdir z (f | d) (압축할_경로) (생성될_zip_경로)`
  * 해제: `/sdir u (대상.zip) (도착폴더)`
* *주의:* 서버 환경에 따라 `unzip` 기능은 완벽히 동작하지 않을 수 있습니다.

## 💻 4. 텍스트 에디터 (파일 읽기 / 쓰기)
서버 스크립트나 설정 파일 코드를 채팅창에서 수정합니다. **대상이 반드시 `file`이어야 합니다.**

### 4-1. 파일 열람 (Read)
* **정식 문법:** `/sdir edit read (파일_경로) [읽을_줄_범위]`
* **단축 문법:** `/sdir e r (파일_경로) [읽을_줄_범위]`
* **출력 스펙:** 500줄 초과 파일은 범위를 지정하지 않으면 출력이 차단됩니다.
* **범위 입력 방식:** 생략(전체 출력), 단일 숫자(예: `5`), 범위(예: `1-10` 또는 `10-1`)

### 4-2. 내용 수정 및 기록 (Write)
지정한 줄의 텍스트를 완전히 덮어쓰거나 지웁니다. 단어 치환(Replace)이 아님에 주의하십시오.
* **정식 문법:** `/sdir edit write (줄_번호) (파일_경로) [수정할_문자열]`
* **단축 문법:** `/sdir e w (줄_번호) (파일_경로) [수정할_문자열]`

#### ⚠️ 에디터 세부 동작 스펙
* **덮어쓰기:** 기존 코드 라인이 `[수정할_문자열]`로 통째로 변경됩니다.
* **내용 삭제:** `[수정할_문자열]`을 입력하지 않으면 해당 줄의 내용이 빈칸(Blank)으로 삭제됩니다.
* **라인 추가:** 파일의 총 줄 수보다 큰 줄 번호를 입력하면 파일의 맨 마지막 줄에 새로운 내용이 추가됩니다.
* **들여쓰기(Tab) 치환:** 인게임 환경의 제약을 우회하기 위해 문자열 내에 **`<~>`** 를 입력하면 시스템이 이를 실제 탭(`\t`)으로 치환하여 저장합니다. (Skript 리로드 시 에러가 발생한다면, 기존 코드의 들여쓰기 횟수와 동일하게 `<~>`를 입력했는지 확인하십시오.)
